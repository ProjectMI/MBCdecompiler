ЗАПУСК ПРОЕКТА

1. Построить нормализованный IR для одного скрипта и сохранить его в файл:
python analyze_mbc.py door1.mbc

По умолчанию IR будет сохранён в:
ir/door1.ir.json

Можно указать свой путь:
python analyze_mbc.py door1.mbc --out out/door1.ir.json

Можно передавать имя и без расширения:
python analyze_mbc.py door1

2. Прогон по всему корпусу .mbc без сохранения IR.
В этом режиме строится IR для каждого скрипта, но на диск он не пишется. Вместо этого формируется краткий отчёт с метриками корпуса и списком аномальных IR:
python analyze_mbc.py --out report.json

СТРУКТУРА

- `IR.py` — построение нормализованного IR, нормализация семантики токенов, агрегирование метрик и поиск аномалий.
- `analyze_mbc.py` — CLI-обвязка с двумя режимами: single-script IR и corpus report.
- `mbc/` — корпус скриптов.
- `ir/` — директория, куда по умолчанию сохраняется IR для отдельного скрипта.
